GeoDCAT-AP 2.0.0

Until GeoDCAT-AP version 1.0.1 [[GeoDCAT-AP-20160802]], property dct:hasPart was used with class dcat:Catalog to specify a service documented in a Catalogue, since no specific property was available in reference vocabularies.

[[VOCAB-DCAT-2]] has defined a specific property dcat:service for this purpose. Therefore it replaces dct:hasPart starting from this version of GeoDCAT-AP.

It is worth noting that dcat:service is a subproperty of dct:hasPart.

Until GeoDCAT-AP version 1.0.1 [[GeoDCAT-AP-20160802]], property dct:subject was used with class dcat:Catalog to specify a category of a Catalogue, since the use of property dcat:theme was restricted to dcat:Dataset's in the version of the W3C Data Catalog Vocabulary available at that time [[VOCAB-DCAT-20140116]].

[[VOCAB-DCAT-2]] has relaxed this domain restriction, so to allow the use of property dcat:theme for any resource in a Catalogue. Therefore it replaces dct:subject starting from this version of GeoDCAT-AP.

It is worth noting that dcat:theme is a subproperty of dct:subject.
