GeoDCAT-AP 2.0.0

Until GeoDCAT-AP version 1.0.1 [[GeoDCAT-AP-20160802]], property rdfs:comment was used with class dcat:Dataset to specify the spatial resolution of a Dataset, since a more specific property was not available in reference vocabularies.

[[VOCAB-DCAT-2]] and [[?SDW-BP]] recommend the use of [[VOCAB-DQV]] for the specification of different types of spatial resolution, via property dqv:hasQualityMeasurement. Moreover, [[VOCAB-DCAT-2]] has defined a specific property to specify spatial resolution as distance in metres, namely, dcat:spatialResolutionInMeters.

Therefore, starting from this version of GeoDCAT-AP, property dcat:spatialResolutionInMeters replaces rdfs:comment when spatial resolution is expressed as distance in metres, whereas property dqv:hasQualityMeasurement replaces rdfs:comment in the other cases.

Property rdfs:comment MAY still be used when spatial resolution is specified as free-text.
