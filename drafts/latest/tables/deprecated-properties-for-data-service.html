GeoDCAT-AP 2.0.0

Until GeoDCAT-AP version 1.0.1 [[GeoDCAT-AP-20160802]], property rdfs:comment was used with class dctype:Service (replaced by class dcat:DataService since this version of GeoDCAT-AP) to specify the spatial resolution of a Data Service, since a more specific property was not available in reference vocabularies.

[[VOCAB-DCAT-2]] and [[?SDW-BP]] recommend the use of [[VOCAB-DQV]] for the specification of different types of spatial resolution, where property dqv:hasQualityMeasurement is used for this purpose. Moreover, [[VOCAB-DCAT-2]] has defined a specific property to specify spatial resolution as distance in metres, namely, dcat:spatialResolutionInMeters.

Therefore, starting from this version of GeoDCAT-AP, property dcat:spatialResolutionInMeters replaces rdfs:comment when spatial resolution is expressed as distance in metres, whereas property dqv:hasQualityMeasurement replaces rdfs:comment in the other cases.

Property rdfs:comment MAY still be used when spatial resolution is specified as free-text.

Until GeoDCAT-AP version 1.0.1 [[GeoDCAT-AP-20160802]], property dct:subject was used with class dctype:Service (replaced by class dcat:DataService since this version of GeoDCAT-AP) to specify a category of a Data Service, since the use of property dcat:theme was restricted to dcat:Dataset's in the version of the W3C Data Catalog Vocabulary available at that time [[VOCAB-DCAT-20140116]].

[[VOCAB-DCAT-2]] has relaxed this domain restriction, so to allow the use of property dcat:theme for any resource in a Catalogue. Therefore it replaces dct:subject starting from this version of GeoDCAT-AP.

It is worth noting that dcat:theme is a subproperty of dct:subject.
